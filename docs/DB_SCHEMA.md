# Database Schema Documentation

## Overview

Busibot uses PostgreSQL 15+ for data persistence. All tables use UUID primary keys and include timestamps for audit trails.

## Tables

### users
Stores user account information.

```sql
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email VARCHAR(255) NOT NULL UNIQUE,
  password_hash VARCHAR(255) NOT NULL,
  first_name VARCHAR(100) NOT NULL,
  last_name VARCHAR(100) NOT NULL,
  company VARCHAR(255),
  phone VARCHAR(20),
  avatar_url VARCHAR(500),
  is_active BOOLEAN DEFAULT true,
  email_verified BOOLEAN DEFAULT false,
  last_login TIMESTAMP,
  created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  deleted_at TIMESTAMP
);

INDEX: email, is_active, created_at
```

### projects
User projects/startups.

```sql
CREATE TABLE projects (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  title VARCHAR(255) NOT NULL,
  description TEXT,
  status VARCHAR(50) DEFAULT 'draft',
  visibility VARCHAR(50) DEFAULT 'private',
  tags TEXT[],
  created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  deleted_at TIMESTAMP
);

INDEX: user_id, status, created_at
```

### ideas
Business ideas generated by or saved by users.

```sql
CREATE TABLE ideas (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  project_id UUID REFERENCES projects(id) ON DELETE SET NULL,
  title VARCHAR(255) NOT NULL,
  description TEXT NOT NULL,
  industry VARCHAR(100),
  sub_industry VARCHAR(100),
  problem TEXT,
  solution TEXT,
  target_market VARCHAR(255),
  market_size VARCHAR(50),
  market_size_value DECIMAL(15, 2),
  validation_status VARCHAR(50),
  validation_score DECIMAL(3, 2),
  feasibility_score DECIMAL(3, 2),
  market_opportunity_score DECIMAL(3, 2),
  founder_market_fit_score DECIMAL(3, 2),
  recommendation VARCHAR(255),
  metadata JSONB,
  created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  deleted_at TIMESTAMP
);

INDEX: user_id, project_id, industry, validation_status, created_at
```

### business_plans
Generated business plans.

```sql
CREATE TABLE business_plans (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  idea_id UUID NOT NULL REFERENCES ideas(id) ON DELETE CASCADE,
  user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  title VARCHAR(255) NOT NULL,
  company_name VARCHAR(255),
  executive_summary TEXT,
  market_analysis TEXT,
  company_description TEXT,
  products_services TEXT,
  business_model TEXT,
  go_to_market_strategy TEXT,
  management_team TEXT,
  competitive_advantage TEXT,
  financial_projections JSONB,
  risk_analysis TEXT,
  funding_requirements DECIMAL(15, 2),
  status VARCHAR(50) DEFAULT 'draft',
  created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  deleted_at TIMESTAMP
);

INDEX: idea_id, user_id, status, created_at
```

### market_research
Market research data and analysis.

```sql
CREATE TABLE market_research (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  idea_id UUID NOT NULL REFERENCES ideas(id) ON DELETE CASCADE,
  user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  market_size VARCHAR(50),
  market_size_value DECIMAL(15, 2),
  growth_rate DECIMAL(5, 2),
  cagr DECIMAL(5, 2),
  market_trends TEXT[],
  customer_segments TEXT[],
  pricing_analysis JSONB,
  competitors JSONB,
  market_drivers TEXT,
  barriers_to_entry TEXT,
  regulatory_considerations TEXT,
  created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  deleted_at TIMESTAMP
);

INDEX: idea_id, user_id, created_at
```

### financial_forecasts
Financial projections and forecasts.

```sql
CREATE TABLE financial_forecasts (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  idea_id UUID NOT NULL REFERENCES ideas(id) ON DELETE CASCADE,
  user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  business_plan_id UUID REFERENCES business_plans(id) ON DELETE SET NULL,
  initial_investment DECIMAL(15, 2),
  monthly_burn_rate DECIMAL(15, 2),
  break_even_month INTEGER,
  runway_months DECIMAL(5, 2),
  assumptions JSONB,
  monthly_projections JSONB,
  annual_projections JSONB,
  revenue_model TEXT,
  unit_economics JSONB,
  created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  deleted_at TIMESTAMP
);

INDEX: idea_id, user_id, created_at
```

### investor_matches
Matched investors and investment opportunities.

```sql
CREATE TABLE investor_matches (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  idea_id UUID NOT NULL REFERENCES ideas(id) ON DELETE CASCADE,
  user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  investor_id UUID,
  investor_name VARCHAR(255),
  investor_type VARCHAR(50),
  industry_focus TEXT[],
  investment_range_min DECIMAL(15, 2),
  investment_range_max DECIMAL(15, 2),
  match_score DECIMAL(3, 2),
  match_reasoning TEXT,
  portfolio_fit_score DECIMAL(3, 2),
  stage_focus VARCHAR(50),
  contact_name VARCHAR(255),
  contact_email VARCHAR(255),
  contact_phone VARCHAR(20),
  website VARCHAR(500),
  notes TEXT,
  status VARCHAR(50),
  created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  deleted_at TIMESTAMP
);

INDEX: idea_id, user_id, match_score, created_at
```

### conversations
Chat conversations between users and AI agents.

```sql
CREATE TABLE conversations (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  idea_id UUID REFERENCES ideas(id) ON DELETE SET NULL,
  title VARCHAR(255),
  context JSONB,
  agent_type VARCHAR(50),
  is_archived BOOLEAN DEFAULT false,
  created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  deleted_at TIMESTAMP
);

INDEX: user_id, idea_id, is_archived, created_at
```

### messages
Individual messages in conversations.

```sql
CREATE TABLE messages (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  conversation_id UUID NOT NULL REFERENCES conversations(id) ON DELETE CASCADE,
  user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  user_message TEXT NOT NULL,
  assistant_response TEXT,
  agent_used VARCHAR(100),
  agent_metadata JSONB,
  tokens_used INTEGER,
  processing_time_ms INTEGER,
  created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);

INDEX: conversation_id, user_id, created_at
```

### pitch_decks
Generated pitch decks.

```sql
CREATE TABLE pitch_decks (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  idea_id UUID NOT NULL REFERENCES ideas(id) ON DELETE CASCADE,
  user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  title VARCHAR(255),
  slides JSONB,
  format VARCHAR(50),
  file_url VARCHAR(500),
  download_count INTEGER DEFAULT 0,
  created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  deleted_at TIMESTAMP
);

INDEX: idea_id, user_id, created_at
```

### audit_logs
Audit trail for all actions.

```sql
CREATE TABLE audit_logs (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES users(id) ON DELETE SET NULL,
  action VARCHAR(255) NOT NULL,
  resource_type VARCHAR(100),
  resource_id UUID,
  old_values JSONB,
  new_values JSONB,
  ip_address VARCHAR(50),
  user_agent VARCHAR(500),
  status VARCHAR(50),
  error_message TEXT,
  created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);

INDEX: user_id, resource_type, resource_id, created_at
```

## Relationships

```

users (1) ─── (N) conversations
users (1) ─── (N) audit_logs

projects (1) ─── (N) ideas

ideas (1) ─── (N) business_plans
ideas (1) ─── (N) market_research
ideas (1) ─── (N) financial_forecasts
ideas (1) ─── (N) investor_matches
ideas (1) ─── (N) pitch_decks
ideas (1) ─── (N) conversations

business_plans (1) ─── (N) financial_forecasts
```

## Migrations

All schema changes use Alembic for version control:

```bash
# Create migration
alembic revision --autogenerate -m "description"

# Apply migrations
alembic upgrade head

# Rollback
alembic downgrade -1
```
